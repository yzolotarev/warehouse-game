# Игра-склад продуктивности - лог решений (design phase)

Проект: геймифицированная таск-система «склад Озон». Ведём по playbook build-with-ai.
Обновлено: 2026-07-10.

## Зафиксированные решения

### Философия
- Работников-скриптов «вместо себя» нет: работник склада = Ярослав на автопилоте (мало сил), мыслитель = он же в режиме блуждания (рефлексия). Одна ось: сил много (доступно всё) / сил мало (только простое или отдых).
- Уровень сил заявляет сам юзер в моменте, система не предсказывает.
- Метрика №1: частота возвращений в систему (прошлые cc-gtd/cc-zfg умерли от редких возвращений: трение входа + непривлекательность).
- Продукт для себя = как продукт для других (нулевой порог входа). Open source.
- ПДВ-петля Коваля (Предвкушение → Действие → Вознаграждение): награда видна ДО действия.

### Архитектура
- Сервер-демон (systemd, всегда работает) владеет БД задач - единственный источник истины.
- Игра = Chrome --app окно поверх сервера; открытие в один жест. Игра читает мир и пишет действия («✓ Сделал») обратно в БД.
- Глобальный хоткей инбокса: **Ctrl+Alt+A**.
- Мобильный вход: MacroDroid на телефоне → служебный TG-чат → воркер забирает в инбокс.
- GCal ВНЕ системы: события со временем человек заносит напрямую в календарь.
- Obsidian: write-only стрелка. Мысль с триажа → markdown в 2nd brain, НО только после того как AI-работник прописал ≥1 вики-ссылку (гейт против заметок-сирот). (Ранее «не сохранять» - юзер развернул решение 10.07, ждёт финального подтверждения.)
- AI-скрипты (gemini-web2api) проектируем В КОНЦЕ, как мостики между готовыми объектами системы.

### Жизненный цикл коробки (задачи)
- Инбокс хранит время + сырой текст, ничего не теряет. Инбокс = средство поведенческой рефлексии (лента следов внимания за день; по данным inbox_dump только ~2% потока - задачи).
- Триаж-веер из инбокса: Фокус ≤5 / Стеллажи (КНП) / Паллета-проект / Ожидание; мусор удаляется при разборке (норма ~30%); мысль → Obsidian через вики-гейт.
- Активные полки отгружают сделанное в Done (Отгрузка). Done лежит на виду до 22:00, затем «фура увозит» (accomplishment, стык с вечерним ревью 21:30).
- [НЕ ПОДТВЕРЖДЕНО юзером, критично] Паллета-проект выставляет один одношаговый шаг в Фокус; сделал шаг → назначь следующий.

### Канон (из NLM: Вотяков 9 списков + ZFG + Аллен)
- Полный набор: Inbox, Backlog, Ожидание, Проекты, Календарь, КНП, Заметки, Справочные, Done.
- Триаж: что это? → делать? (30% мусор) → мне? → сейчас? → мутная = проект → ≤2 мин = сделать сразу.
- Два столпа (убери один - развал за 2 недели): ежедневный разбор 10-15 мин + Week Review ~2ч.
- Старение: Фокус >3 дней без прогресса → Отложка/удалить; проект без движения месяц → заморозка.
- Крючки Аллена: только первичный сбор (6-15ч, 100-300 задач) → игровой onboarding «день открытия склада».

### Механики из реального склада (сессия учёбы WMS)
- Терминал (ТСД): режим «сил мало» = одна коробка + один вопрос на экране, не весь склад.
- Арбитраж: мутные коробки не блокируют разбор - уезжают в очередь арбитража для мыслителя (сил много).
- WMS = наш сервер-демон; Оператор = юзер-автопилот.

### Игровой слой (решено 10.07)
- Действия → Очки → Магазин → Развитие склада (новые зоны, красота).
- Цели (TELOS) = витрина, НЕ механика: не фильтруют вход, не множат очки; витрина прогресса показывает очки, итоги ревью, движение к целям. ПДВ-петля (G) питает цели психологически.
- Фильтра на входе НЕТ: в инбокс падает всё подряд, чистка на триаже вручную («пока что»).
- Ритуалы = игровые события. Склад зовёт сам, двухступенчато: сначала мягкая подсветка (можно игнорировать), затем СТОПОР - склад останавливается, коробки не двигаются, пока не проведён разбор. Физика мира вместо штрафов.
- {Утро, Фура 22:00} = автоматика склада; {Вечерний разбор + Зеркало дня, Недельная пересборка} = ритуалы человека, за них награда.
- Зеркало дня: лента инбокса за день показывается внутри вечернего разбора перед триажем.

### Паллета-проект (решено 10.07)
- Паллета = критерий готовности + план шагов; план выставляет одношаговые шаги в Фокус (подтверждено юзером).
- Разморозка - только вручную, человеком.
- БАТЧИНГ: игра стимулирует очками брать в Фокус пачку шагов одной паллеты (бонус за серию, виден заранее по ПДВ) - против «4 разрозненных задачи, нет погружения».
- Петля «сделал шаг → назначь следующий» - правило ревью, не жёсткая автоматика.
- Открыто (решить на плане): пачка из 3 шагов = 3 слота Фокуса или 1 слот-очередь.

### Ожидание (решено 10.07)
- Коробка сереет при помещении. По таймеру напоминания загорается (glow): «пора чекнуть».
- Расписание - функция: интервалы растут (1-3-7-14) или сжимаются (14-7-3-1), либо ручное «каждые n дней».
- Чек светящейся коробки - три кнопки: done (сразу в отгрузку) / wait more (таймер дальше) / delete.

### Мысли и справочные (решено 10.07)
- Мысли НЕ живут на складе. Жизнь мысли: инбокс → либо удалена, либо в Obsidian (через вики-гейт работника). Полки мыслей нет.
- Справочные - предположение: аналогично в Obsidian, склад не хранит (юзер явно не подтверждал).

### Интерфейс
- Мир игры = атмосфера/прогресс; работа с задачами = чистая панель по клику на зону («идеальный Obsidian»). Ховер-тултипы - деталь, не основной способ.
- Изометрия, зоны: приёмка (грузовик), стеллажи, фокус, отгрузка, кофейня (отдых работников).
- Графика: прототип-вкус одобрен частично; настоящая версия = готовые ассеты (Kenney CC0) + движок (PixiJS/Phaser). Процедурный картон юзер оценил как «скверный».
- Хотелки юзера: поворот камеры, анимации/звуки, отдых работников (сделано в proto v2).

### Здоровье склада (карта 10.07 поздно; строить с игровым миром)
- Спорт/творчество/отдых = действия «n раз в неделю», датчик = галочка юзера (self-report, календарь/таймер).
- Пропуски → чистота склада падает → насекомые/грязь копятся по чуть-чуть (визуальный градиент, не штраф).
- Уборка: ПО ВЫБОРУ (кастомизация) - «галочка активности = работник метёт» или «уборка = отдельное дело».
- Кофейня = узел отдыха и для работников, и для юзера («тело и душа»).

### Ответы канона 11.07 (NLM: Sung 396c1e13 + Вотяков bf109a9c; юзер устал, стрелки рисовал агент)
- «Извлечь проекты обратно?» - НЕТ. Вотяков: при внедрении ВСЁ (включая проекты старой системы) обязано пройти через корзину - миграция 10.07 сделана точно по канону. Тревога «системе не доверяю» = штатная фаза, длится 1-2 месяца, лечится только использованием (довёл дело до конца через систему → +доверие). Заметки/справочные в корзину не сгребать - остаются в Obsidian (наш пункт 3 канон подтверждает).
- Вопрос №1 (2-й месяц): мотивация = 20-30% поведения и выветрится точно. Выживание = cue-dependent, не motivation-dependent: наши зов 21:30 / фура 22:00 / баннер пересборки - это и есть cues, их беречь. Добавить scripted action для худшего дня: «устал → всё равно открыть склад на 2 мин и отметить 1 ⭐ на завтра». Санг: первые версии системы полны band-aid-костылей - это норма, снимать постепенно.
- Вопрос №5 (утро): решений утром НОЛЬ (PFC-бюджет). Канон: взгляд в календарь (якоря дня) → готовый Фокус 5-7, собранный с вечера, с подсвеченными 1-2 ⭐ (frontloading). Утренний экран = «вот твои якоря, вот твоя ⭐» - не спрашивать «что будем делать?».
- Две ошибки, убивающие систему за недели: пропуск ежедневного разбора корзины + пропуск week review. Обе механики у нас уже встроены (фура + /review) - не выпиливать при редизайнах.
- Правило изменений: списки/формы кастомизируй, ДЕРЕВО РАЗБОРА ИНБОКСА не трогай (в нём нет дырок - это суть системы). Новому устройству давать 2-3 дня обкатки, не переделывать по три раза на дню.

### Словарь (уточнён 10.07 вечером)
- **Товар** = одна задача (единица, приезжает россыпью с приёмки).
- **Коробка** = несколько товаров одного контекста (тарник, контекстуализация). По складу ездят коробками.
- В БД таблица пока называется boxes = товары; переименование отложено, в UI использовать «товар».

### ЖЕЛЕЗНОЕ правило интерфейса (юзер, 10.07)
**Один объект на экран. Скролла не существует.** Любой список = очередь по одному (счётчик «осталось N») или листание ‹ › по одной карточке. Применять ко ВСЕМ будущим экранам.

### Правило двух языков (юзер, 10.07)
Каждый складской термин в UI подписывать нормальным языком таск-менеджмента: «Паллетная зона = Проекты», «Терминал = разбор инбокса», «Ожидание = делегированное/ждёшь ответа». Метафора - вкус, термины - ясность.

- 6 кнопок убиты: узкая окрестность внимания выдерживает max 2-3.
- Решение = глубина вместо ширины: канон-дерево по одному бинарному вопросу на экран (2 кнопки), 3-4 экрана на товар.
- Позже (этап скриптов): предсортировка ИИ - готовый ярлык-маршрут, счастливый путь «да / не так» = 1-2 кнопки.
- Апгрейд 10.07 вечер: не «товар через все вопросы», а КОНВЕЙЕРНЫЕ ПРОХОДЫ - один вопрос через всю пачку (5 проходов), один режим мышления на пачку. Юзер: «уже лучше, мне нрав» (живой тест).
- Фильтра на входе по-прежнему НЕТ (падает всё подряд).

- Паллетная зона: «📖 Заметка проекта» (просмотр + список файлов папки), строка «дописать контекст» (нет заметки → файл создаётся сам по названию), «📎 Привязать» (список файлов из /obsidian_files).
- Автопривязка при оформлении: текст «📁 Проект (Obsidian): X» → fuzzy-match файла (_norm: lower/ё/тире, пересечение токенов ≥.5).
- Одноразовый импорт 11.07: паллеты 2-5 привязаны (vacancies_list.md, Военкомат/, Казино.md, Сцена допроса.md), титулы очищены от префикса. Папка-проект: дозапись идёт в Склад-заметки.md внутри неё.
- Работники мира: звуки убраны («надоедают»), поведение честное - суета только при непустом инбоксе, с коробкой в кофейню нельзя, при пустом складе стоят/пьют кофе (юзер: «должно синхронизироваться с реальной жизнью»).

### Тотальный журнал (достроен 11.07 по вопросу юзера)
До этого «полное логирование» было честным только для движений (moves) и очков (points). Теперь таблица **events** - единый поток: inbox_add, move (с очками), star, wait_more, pallet_new/step/freeze/delete, review_finish, rest_start/finish, sched_morning/truck/autofreeze + **ui_triage_answer** (каждый ответ на каждый вопрос триажа с текстом коробки - главный датасет для ИИ-предсортировки) и ui_triage_undo (сигнал неуверенных решений). Клиент шлёт через sendBeacon POST /event (kind получает префикс ui_). Чтение: GET /events?limit&kind. НЕ логируются: просмотры страниц и отбитые стопором попытки (откат транзакции при 423 съедает запись - осознанный компромисс).
