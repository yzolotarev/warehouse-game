# warehouse-game

## ИИ-работник склада (протокол)

В начале рабочей сессии выполни `./bin/wh queue` - это окно ИИ-работника на
живом складе Ярослава (сервер на 127.0.0.1:8091, systemd-юнит `warehouse-server`).

- **Мои задачи** = только то, что выдаёт `wh queue`: до 5 коробок, размеченных
  человеком как «Робот» на конвейере /robot_page. Никакого гадания по тексту.
- В окне есть задачи → сказать юзеру: «в окне N задач, беру?» Брать после «да»
  (или сразу, если он сам попросил поработать на складе).
- Сделал задачу → `./bin/wh report ID "что сделано, где результат"`.
  Коробка выпадает из окна, втекает следующая. Коробку НЕ закрывать и не
  двигать - «✅ Сделал» жмёт человек, увидев отчёт на карточке фокуса.
- Задача мутная → отчёт вида «понял так: ..., верно?» вместо самодеятельности.
  Задача-проект → один конкретный шаг + «дальше предлагаю так», не комбайн.
- Побочная мысль/находка → `./bin/wh add "текст"` (уйдёт в инбокс на триаж).
- `./bin/wh focus` - очередь фокуса человека (посмотреть контекст, не трогать).

## Код

- Сервер: `server/warehouse_server.py` (FastAPI + sqlite, всё в одном файле),
  страницы - `server/*.html`. Граф кода: `graphify-out/`.
- Новые UI-строки дублировать в словарь PAIRS в `server/i18n.js` (RU/EN).
- После правок сервера: `systemctl --user restart warehouse-server`.
